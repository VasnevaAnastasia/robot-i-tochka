    Задачи для первого релиза

    TC-01 Регистрация в программе лояльности

    Use case: UC-01 Регистрация в программе лояльности

    Номер ТС Название сценария Тип Сценарий
    TC-01.01 Успешная регистрация нового пользователя Позитивный Предусловие: Пользователь не зарегистрирован.

    Шаги:
    1. Нажать «Регистрация».
    2. Ввести корректный номер.
    3. Получить и ввести верный код.
    4. Ввести пароль.
    5. Нажать «Сохранить».

    Ожидаемый результат: Система создаёт учётную запись, авторизует пользователя, отображает личный кабинет.

    Постусловие: Учетная запись создана, пользователь авторизован.
    TC-01.02 Пользователь уже зарегистрирован Альтернативный Предусловие: Пользователь уже есть в системе.

    Шаги:
    1. Нажать «Регистрация»
    2. Ввести номер, уже зарегистрированный в системе.

    Ожидаемый результат: Система выводит сообщение «Вы уже зарегистрированы», предлагает ввести пароль.

    Постусловие: Регистрация прекращена, переход к сценарию авторизации.
    TC-01.03 Повторный запрос кода (если SMS не пришло) Альтернативный Предусловие: Пользователь не зарегистрирован, код не был получен.

    Шаги:
    1. Ввести номер.
    2. Нажать «Получить код».
    3. Дождаться таймера.
    4. Нажать «Отправить код еще раз».
    5. Ввести полученный код.
    6. Продолжить регистрацию.

    Ожидаемый результат: Код отправлен повторно, регистрация завершается успешно.

    Постусловие: Учетная запись создана.
    TC-01.04 Ввод некорректного кода подтверждения Негативный Предусловие: Пользователь получил код, но ввел неверный.

    Шаги:
    1. Ввести неправильный код подтверждения.

    Ожидаемый результат: Система показывает сообщение об ошибке «Неверный код».

    Постусловие: Пользователь остаётся на экране ввода кода.
    TC-01.05 Ввод номера в неправильном формате Негативный Предусловие: Пользователь находится на форме регистрации.

    Шаги:
    1. Ввести номер телефона в неправильном формате (буквы, символы, короткий номер).

    Ожидаемый результат: Система выдаёт сообщение об ошибке.

    Постусловие: Регистрация не начата.
    TC-01.06 Несовпадение паролей при создании Негативный Предусловие: Пользователь дошёл до шага создания пароля.

    Шаги:
    1. Ввести пароль.
    2. Повторно ввести другой пароль. 3. Нажать «Сохранить».

    Ожидаемый результат: Система показывает ошибку «Пароли не совпадают».

    Постусловие: Регистрация не завершена.
    TC-01.07 Пропуск необязательных данных Позитивный Предусловие: Пользователь на экране ввода пароля для ЛК.

    Шаги:
    1. Не заполнять имя, город, дату рождения.
    2. Ввести только пароль.
    3. Сохранить.

    Ожидаемый результат: Система сохраняет профиль, завершает регистрацию.

    Постусловие: Учетная запись создана.
    TC-01.08 Пропуск обязательных данных (не введён пароль) Негативный Предусловие: Пользователь на экране ввода пароля для ЛК.

    Шаги:
    1. Оставить поле пароля пустым.
    2. Нажать «Сохранить».

    Ожидаемый результат: Система отображает ошибку «Введите пароль».

    Постусловие: Регистрация не завершена, пользователь остаётся на экране ввода.

    TC-02 Авторизация в программе лояльности

    Use case: UC-02 Авторизация в программе лояльности

    Номер ТС Название сценария Тип Сценарий
    TC-02.01 Успешная авторизация Позитивный Предусловие: Пользователь не авторизован в системе.

    Шаги:
    1. Пользователь нажимает «Войти в личный кабинет».
    2. Вводит корректный номер телефона и верный пароль.
    3. Нажимает кнопку «Войти».

    Ожидаемый результат: Система находит пользователя и успешно авторизует его. Открывается личный кабинет.

    Постусловие: Пользователь авторизован.
    TC-02.02 Неверный пароль Негативный Предусловие: Пользователь не авторизован в системе.

    Шаги:
    1. Пользователь нажимает «Войти в личный кабинет».
    2. Вводит корректный номер телефона и неверный пароль.
    3. Нажимает «Войти».

    Ожидаемый результат: Система сообщает: «Пароль не корректный, попробуйте ввести еще раз». Отображается кнопка «Восстановить пароль».

    Постусловие: Пользователь не авторизован.
    TC-02.03 Превышен лимит попыток ввода пароля Негативный Предусловие: Пользователь не авторизован в системе.

    Шаги:
    1. Вводит номер телефона и трижды неверный пароль.
    2. После третьей попытки система блокирует вход.

    Ожидаемый результат: Система сообщает: «Достигнуто максимальное количество попыток». Система блокирует попытку ввести пароль на 5 минут.

    Постусловие: Пользователь не авторизован.
    TC-02.04 Авторизация с номером которого нет в системе Негативный Предусловие: Пользователь не авторизован в системе.

    Шаги:
    1. Вводит незарегистрированный номер телефона и любой пароль.
    2. Нажимает «Войти».

    Ожидаемый результат: Система сообщает, что пользователь с таким номером не найден.

    Постусловие: Пользователь не авторизован.
    TC-02.05 Пустое поле пароля Негативный Предусловие: Пользователь не авторизован в системе.

    Шаги:
    1. Вводит номер телефона, не заполняет поле пароля.
    2. Нажимает «Войти».

    Ожидаемый результат: Система сообщает: «Поле пароля не может быть пустым».

    Постусловие: Пользователь не авторизован.
    TC-02.06 Прерывание авторизации пользователем Альтернативный Предусловие: Пользователь начал процесс авторизации.

    Шаги:
    1. На любом этапе нажимает кнопку «Назад».

    Ожидаемый результат: Система возвращает пользователя на предыдущий экран или завершает процесс входа.

    Постусловие: Пользователь не авторизован.

    TC-03 Восстановление пароля

    Use case: UC-03 Восстановление пароля от ЛК

    Номер ТС Название сценария Тип Сценарий
    TC-03.01 Успешное восстановление пароля Позитивный Предусловие: Пользователь зарегистрирован и не авторизован в системе.

    Шаги:
    1. На экране авторизации нажимает «Восстановить пароль».
    2. Вводит корректный номер телефона.
    3. Получает SMS и вводит правильный код подтверждения.
    4. Вводит новый пароль и нажимает «Сохранить».

    Ожидаемый результат: Пароль обновлён, пользователь может войти в ЛК с новым паролем.

    Постусловие: Новый пароль сохранён.
    TC-03.02 Неверный код подтверждения Негативный Предусловие: Пользователь на экране ввода кода после запроса восстановления пароля.

    Шаги:
    1. Вводит неверный код из SMS.
    Ожидаемый результат: Система сообщает: «Неверный код подтверждения».

    Постусловие: Пароль не изменён.
    TC-03.03 Номер телефона, не зарегистрированный в системе Негативный Предусловие: Пользователь не авторизован.

    Шаги:
    1. Нажимает «Восстановить пароль».
    2. Вводит номер телефона, не зарегистрированный в системе.

    Ожидаемый результат: Система сообщает: «Пользователь с таким номером не найден».

    Постусловие: Пароль не изменён.
    TC-03.04 Пропуск обязательного ввода пароля Негативный Предусловие: Пользователь дошёл до шага ввода нового пароля.

    Шаги:
    1. Не вводит пароль и нажимает «Сохранить».

    Ожидаемый результат: Система сообщает: «Поле пароля не может быть пустым».

    Постусловие: Пароль не изменён.
    TC-03.05 Успешный повторный запрос кода подтверждения Альтернативный Предусловие: Пользователь не получил SMS с кодом.

    Шаги:
    1. Дожидается окончания таймера.
    2. Нажимает кнопку «Отправить код еще раз».
    3. Получает и вводит корректный код.

    Ожидаемый результат: Система принимает новый код, переход к шагу ввода нового пароля.

    Постусловие: Пароль может быть обновлён.
    TC-03.06 Прерывание процесса восстановления пароля Альтернативный Предусловие: Пользователь начал восстановление пароля.

    Шаги:
    1. На любом шаге нажимает «Назад».

    Ожидаемый результат: Система прекращает восстановление пароля и возвращает на предыдущий экран.

    Постусловие: Процесс восстановления пароля отменён.

    TC-04 Просмотр личного кабинета

    Use case: UC-04 Просмотр личного кабинета программы лояльности

    Номер ТС Название сценария Тип Сценарий
    TC-04.01 Успешный просмотр личного кабинета Позитивный Предусловие: Пользователь авторизован в системе.

    Шаги:
    1. Нажимает иконку личного кабинета.
    2. Система открывает страницу ЛК с персональными данными (имя, бонусы, история заказов и пр.).

    Ожидаемый результат: Личный кабинет отображается корректно.

    Постусловие: Пользователь просмотрел свой ЛК.
    TC-04.02 Ошибка при загрузке данных Негативный Предусловие: Пользователь авторизован, но возникает ошибка при обращении к серверу.

    Шаги:
    1. Нажимает иконку ЛК.
    2. Система не может загрузить данные из-за ошибки (например, 500).

    Ожидаемый результат: Отображается сообщение: «Не удалось загрузить данные. Попробуйте позже».

    Постусловие: ЛК не отображён.
    TC-04.03 Отображение пустых/неполных данных Негативный Предусловие: Пользователь авторизован, но данные профиля неполные (например, отсутствует имя).

    Шаги:
    1. Открывает ЛК.

    Ожидаемый результат: Отображаются поля без данных.

    Постусловие: Пользователь просмотрел ЛК.
    TC-04.04 Просмотр после выхода из системы Негативный Предусловие: Пользователь разлогинился или сессия истекла.

    Шаги:
    1. Нажимает иконку ЛК после завершения сессии.

    Ожидаемый результат: Система перенаправляет на экран авторизации.

    Постусловие: Просмотр ЛК невозможен без входа.
    TC-04.05 Прерывание просмотра личного кабинета Альтернативный Предусловие: Пользователь авторизован, начал просмотр ЛК.

    Шаги:
    1. Нажимает кнопку «Назад» или закрывает экран.

    Ожидаемый результат: Система возвращает на предыдущий экран.

    Постусловие: Пользователь покинул экран ЛК.

    TC-12 Просмотр истории заказов

    Use case: UC-12 Просмотр истории заказов

    Номер ТС Название сценария Тип Сценарий
    TC-12.01 Успешный просмотр списка заказов Позитивный Предусловие: Пользователь авторизован, в системе есть заказы.

    Шаги:
    1. Переходит в раздел «Заказы».
    2. Система отображает список заказов за последние 3 года.
    3. Пользователь просматривает список.

    Ожидаемый результат: Заказы отображаются с датой, статусом, составом.

    Постусловие: Пользователь просмотрел список заказов.
    TC-12.02 У пользователя нет заказов Альтернативный Предусловие: Пользователь авторизован, у него нет заказов в системе.

    Шаги:
    1. Переходит в раздел «Заказы».

    Ожидаемый результат: Система отображает сообщение «Вы еще ничего не заказали».

    Постусловие: Пользователь получил корректную информацию об отсутствии заказов.
    TC-12.03 Ошибка при загрузке заказов Негативный Предусловие: Пользователь авторизован, происходит ошибка сервера или сети при загрузке заказов.

    Шаги:
    1. Переходит в раздел «Заказы».
    2. Система не может получить список заказов.

    Ожидаемый результат: Система отображает сообщение об ошибке: «Не удалось загрузить заказы. Попробуйте позже».

    Постусловие: Список заказов не загружен.
    TC-12.04 Просмотр заказов после выхода из ЛК Негативный Предусловие: Пользователь вышел из системы до перехода в раздел «Заказы».

    Шаги:
    1. Пытается открыть раздел «Заказы».

    Ожидаемый результат: Система перенаправляет пользователя на экран авторизации.

    Постусловие: Просмотр заказов недоступен без входа в систему.
    TC-12.05 Прерывание процесса просмотра Альтернативный Предусловие: Пользователь начал просмотр заказов.

    Шаги:
    1. Переходит на предыдущий экран или закрывает раздел.

    Ожидаемый результат: Система возвращает пользователя на предыдущую страницу.

    Постусловие: Просмотр заказов завершён пользователем вручную.

    TC-14 Отмена заказа

    Use case: UC-14 Отмена заказа

    Номер ТС Название сценария Тип Сценарий
    TC-14.01 Успешная отмена оплаченного заказа Позитивный Предусловие: Пользователь авторизован, заказ оплачен картой, открыт экран заказа.

    Шаги:
    1. Нажимает «Отменить заказ».
    2. Подтверждает действие.
    3. Система отправляет запрос на возврат в платёжную систему — успех.
    4. Меняет статус заказа на «Отменен».
    5. Уведомляет сервис роботов.
    6. Показывает уведомление об отмене и возврате средств.
    7. Номер заказа исчезает с табло.

    Ожидаемый результат: Заказ отменен, возврат оформлен.

    Постусловие: Заказ отменен, возврат инициирован
    TC-14.02 Отмена неоплаченного заказа Альтернативный Предусловие: Пользователь авторизован, заказ не был оплачен.

    Шаги:
    1. Нажимает «Отменить заказ».
    2. Подтверждает действие.
    3. Система меняет статус на «Отменен».
    4. Уведомляет сервис роботов.
    5. Отображает уведомление об отмене.

    Ожидаемый результат: Отмена завершена без возврата средств.

    Постусловие: Заказ отменен.
    TC-14.03 Пользователь отказался от отмены Альтернативный Предусловие: Пользователь авторизован, открыт заказ.

    Шаги:
    1. Нажимает «Отменить заказ».
    2. Система запрашивает подтверждение.
    3. Пользователь нажимает «Отмена» или закрывает окно.

    Ожидаемый результат: Заказ остается активным, система возвращает на карточку заказа.
    Постусловие: Заказ не изменен.
    TC-14.04 Ошибка при возврате денег Альтернативный Предусловие: Пользователь авторизован, заказ был оплачен картой, открыт заказ.

    Шаги:
    1. Нажимает «Отменить заказ».
    2. Подтверждает действие.
    3. Система отправляет запрос в платёжную систему — получена ошибка.

    Ожидаемый результат: Отображается ошибка отмены, предложено обратиться к менеджеру.

    Постусловие: Заказ не отменен, средства не возвращены.
    TC-14.05 Прерывание процесса отмены заказа Альтернативный Предусловие: Пользователь вышел из системы / сессия устарела, открыт заказ.

    Шаги:
